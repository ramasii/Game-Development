# Skill: Deconstruct Mechanics Using Critical Vocabulary
#game-design #analysis

## Kapan Harus Menggunakan
Gunakan skill ini ketika ingin membedah game kompetitor, menganalisis masalah mekanik (seperti loop/meta yang rusak), atau mendesain elemen inti game baru menggunakan terminologi formal game design agar komunikasi tim lebih presisi.

## Deskripsi
Membedah mekanik game ke dalam komponen fundamental untuk memahami ruang kemungkinan (*possibility space*) dan sifat deterministik permainan.

## Alur Kerja (Workflow)
1. **Map Game State & Possibility Space:** Identifikasi data tersimpan dan variasi tindakan pemain.
2. **Define Determinism & Transitivity:** Tentukan apakah sistem murni skill (Deterministic) atau probabilitas (Stochastic), serta hubungan linear (Transitive) atau melingkar (Intransitive).
3. **Audit Feedback Loops:** Cek efek snowballing (Positive) dan penyeimbangnya (Negative).
4. **Classify Information & Curves:** Tentukan batasan informasi pemain dan tipe kurva pembentuk progres.

## 🔗 Hubungan Keterkaitan
- [[Identify Core Loops]] — Memberikan ruang sekuensial tempat kosakata ini diaplikasikan.
- [[Apply Balance Foundations]] — Fondasi analisis sebelum memutar roda gigi penyeimbangan.
